Variables and Data Types

Variables are containers for storing values in Java programs.

Primitive data types:
  • byte, short, int, long - Integer types
  • float, double - Decimal types
  • boolean - True/false values
  • char - Single characters
public class Variables {
    public static void main(String[] args) {
        int age = 25;
        double salary = 50000.50;
        boolean isEmployed = true;
        char grade = 'A';

        System.out.println("Age: " + age);
        System.out.println("Salary: " + salary);
        System.out.println("Employed: " + isEmployed);
        System.out.println("Grade: " + grade);
    }
}

Java Operators

Operators are symbols used to perform operations on variables and values.

Types of operators:
  • Arithmetic: +, -, *, /, %
  • Comparison: ==, !=, <, >, <=, >=
  • Logical: &&, ||, !
  • Assignment: =, +=, -=, *=, /=
public class Operators {
    public static void main(String[] args) {
        int a = 10, b = 5;

        System.out.println("a + b = " + (a + b));
        System.out.println("a > b = " + (a > b));
        System.out.println("a == b = " + (a == b));

        boolean result = (a > b) && (b > 0);
        System.out.println("Logical AND: " + result);
    }
}

Control Structures

Control structures direct the flow of program execution based on conditions and loops.

Control structures:
if-else, switch, while, for, do-while 
public class ControlStructures {
    public static void main(String[] args) {
        int score = 85;

        // if-else statement
        if (score >= 90) {
            System.out.println("Grade: A");
        } else if (score >= 80) {
            System.out.println("Grade: B");
        } else {
            System.out.println("Grade: C");
        }

        // for loop
        for (int i = 1; i <= 5; i++) {
            System.out.println("Number: " + i);
        }
    }
}

Methods in Java

Methods are blocks of code that perform specific tasks and can be reused throughout the program.

Advantages of methods:
  • Code reusability
  • Modularity and organization
  • Easy debugging and testing
  • Parameter passing and return values
public class Methods {
    public static void main(String[] args) {
        int result = add(15, 25);
        System.out.println("Sum: " + result);

        greetUser("Alice");
    }

    public static int add(int x, int y) {
        return x + y;
    }

    public static void greetUser(String name) {
        System.out.println("Hello, " + name + "!");
    }
}

Object-Oriented Programming

Classes are blueprints for creating objects. Objects are instances of classes with properties and behaviors.

OOP Principles:
Encapsulation, Inheritance, Polymorphism, Abstraction 
public class Person {
    private String name;
    private int age;

    // Constructor
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Getter methods
    public String getName() { return name; }
    public int getAge() { return age; }

    public void displayInfo() {
        System.out.println("Name: " + name + ", Age: " + age);
    }

    public static void main(String[] args) {
        Person person = new Person("John", 30);
        person.displayInfo();
    }
}

Inheritance

Inheritance allows a class to inherit properties and methods from another class, promoting code reusability.

Inheritance concepts:
  • Superclass: Parent class
  • Subclass: Child class
  • extends keyword: Establishes inheritance
  • super keyword: Refers to parent class
class Animal {
    protected String species;

    public void eat() {
        System.out.println("The animal eats");
    }
}

class Dog extends Animal {
    private String breed;

    public Dog(String breed) {
        this.breed = breed;
        this.species = "Canine";
    }

    public void bark() {
        System.out.println("The dog barks");
    }

    public static void main(String[] args) {
        Dog myDog = new Dog("Golden Retriever");
        myDog.eat();  // Inherited method
        myDog.bark(); // Own method
    }
}

Interfaces

Interfaces define contracts that implementing classes must follow, supporting multiple inheritance of type.

Interface features:
Abstract methods, default methods, static methods, constants 
interface Printable {
    void print(); // Abstract method

    default void scan() {
        System.out.println("Scanning document...");
    }
}

class Document implements Printable {
    private String content;

    public Document(String content) {
        this.content = content;
    }

    @Override
    public void print() {
        System.out.println("Printing: " + content);
    }

    public static void main(String[] args) {
        Document doc = new Document("Important Report");
        doc.print();
        doc.scan();
    }
}

Exception Handling

Exception handling provides a way to handle runtime errors gracefully, maintaining program flow.

Exception types:
  • Checked: Must be handled at compile time
  • Unchecked: Runtime exceptions
  • Error: System-level problems
public class ExceptionHandling {
    public static void main(String[] args) {
        try {
            int[] numbers = {1, 2, 3};
            System.out.println(numbers[5]); // This will cause an exception
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Error: " + e.getMessage());
        } finally {
            System.out.println("Cleanup operations");
        }

        // Division by zero example
        try {
            int result = divide(10, 0);
        } catch (ArithmeticException e) {
            System.out.println("Cannot divide by zero!");
        }
    }

    public static int divide(int a, int b) {
        return a / b;
    }
}
